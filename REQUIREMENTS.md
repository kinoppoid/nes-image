# NES Slideshow — Requirements

## 概要

BMPファイルをNES用スライドショーROMに変換するプロジェクト。
複数の画像をファミコン実機またはエミュレータで表示し、コントローラで切り替える。
枚数は可変（`NUM_IMAGES` を自動生成）。NROM-256（32KB PRG-ROM）に収まる枚数が上限（目安10枚）。

---

## ハードウェア仕様（ターゲット）

| 項目 | 内容 |
|------|------|
| マッパー | 0（NROM-256） |
| PRG-ROM | 32KB（$8000–$FFFF） |
| CHR | CHR-RAM 8KB（CHR-ROMなし、実行時に書き込み） |
| WRAM/SRAM | 8KB（$6000–$7FFF）、バッテリー不要 |
| ミラーリング | 水平 |

実機カートリッジには PRG-ROM（27C256相当）と SRAM（6264相当）が必要。
PPU CHR-RAM はファミコン本体内蔵のものを使用。

---

## 画像仕様

| 項目 | 内容 |
|------|------|
| 入力 | 任意解像度、24bpp BMP（デフォルト: `../src-images/*.bmp`） |
| 枚数 | 任意（`--max-images N` で制限可） |
| 出力解像度 | 256×160 px（32×20タイル）を基本とし、タイル超過時は自動縮小 |
| 画面配置 | タイル行 4–23（上下に空白行） |
| スケーリング | ボックスフィルタ（面積平均）で縮小 |
| 自動縮小 | タイル超過時に段階的縮小（256×160 → 224×144 → 192×128 → 160×112 → 128×96） |
| 縮小時配置 | 下端揃え（上部に余白）、余白は背景色で塗りつぶし |

## 色変換パイプライン

### 1. パレットクラスタリング（k-means）

- 画像を 16×16 px（2×2タイル）ブロック 160個に分割
- 各ブロックの特徴量：**CIE L\*a\*b\* 彩度加重平均**（3次元）
  - RGB空間より肌色とグレーを明確に分離できる
  - 彩度（chroma）が高いピクセルほど重みを大きくする
  - 肌色ピクセル（NES_SKIN: `$26,$27,$35,$36,$37`）は重みを3倍にしてクラスタを分離
- k=4 でクラスタリング → 4グループ × 1パレット

### 2. NESパレット選択

- 各クラスタのピクセル群から最適な NES 4色を貪欲法で選択
- 彩度加重誤差で評価（色の濃いピクセルが支配的になる）
- スロット0：背景色（左上ピクセルが画像全体の5%以上を占める場合はその色、それ以外は最も暗い頻出色）

### 3. 緑レスキュー

- 4パレット全てに緑系NES色（`$19`–`$2B`）が含まれない場合のみ発動
- 緑ピクセル（画像全体で10px以上）が存在するとき、最も緑ピクセルを多く持つクラスタのパレットに緑色を強制挿入
- 挿入時は最も損失の少ないスロットを置換（肌色スロットは保護）
- 目的：キャラクターの緑色の目がグレーになることを防ぐ
- 緑が自然に入るパレットがある場合は自動スキップ
- `--no-green-rescue` フラグまたは `make GREEN_RESCUE=0` で無効化可能

### 4. 肌色レスキュー

- 肌色ピクセル（NES_SKIN）が50px以上あり、4パレットに肌色が含まれない場合に発動
- 最も肌色ピクセルを多く持つクラスタのパレットに肌色を強制挿入

### 5. Atkinson ディザリング

- 8×8タイル単位でディザリング
- 誤差を6近傍に 1/8 ずつ分散（合計75%）
- Floyd-Steinberg より固体領域が増え圧縮効率が向上

### 6. タイル重複排除

- 640タイル位置（32×20）を最大256スロットに割り当て
- スロット0：ブランクタイル（全ゼロ）予約
- 256スロット超過時：自動縮小にフォールバック（スケールステップを順に試行）
- 最終ステップでも超過した場合は最近傍スロットにマッピング

## データ圧縮

| データ種別 | 圧縮方式 | 備考 |
|-----------|---------|------|
| CHR タイルデータ | LZE（LZSS系） | 4096B → 平均約2700B |
| ネームテーブル | LZE | 640B → 平均約340B |
| パレット | 非圧縮 | 16B × N枚 |
| アトリビュート | 非圧縮 | 40B × N枚 |

LZEデコーダはZP $1A–$21を使用（CC65ランタイムと非衝突）。

**実行時展開先：**
- CHR：WRAM $6000–$6FFF（4KB）→ PPU CHR-RAM $0000–$0FFF へDMA
- ネームテーブル：WRAM $7000–$72FF（640B）→ PPU ネームテーブルへ書き込み

---

## ROM レイアウト（PRG-ROM 32KB）

| セグメント | 内容 |
|-----------|------|
| `STARTUP`（$8000–） | リセットハンドラ、iNESヘッダ |
| `CODE` | main.c、LZEデコーダ、PPUユーティリティ |
| `RODATA` | 圧縮CHR×N、圧縮NT×N、パレット、アトリビュート |
| `VECTORS`（$FFFA–$FFFF） | NMI・Reset・IRQベクタ |

---

## 操作仕様

| ボタン | 動作 |
|-------|------|
| A または Right | 次の画像へ（最後→最初に戻る） |
| B または Left | 前の画像へ（最初→最後に戻る） |

エッジ検出（押した瞬間のみ反応）。VBlank待ちでスロットリング。

## ビルド環境

| ツール | 用途 |
|-------|------|
| `cc65` | C→アセンブリ変換 |
| `ca65` | アセンブラ |
| `ld65` | リンカ |
| `/usr/share/cc65/lib/nes.lib` | CC65ランタイムライブラリ |
| `python3` | `tools/make_chr.py`（画像変換） |
| `lze.py` | LZEエンコーダ（`../../88image/lze.py`） |

```
sudo apt install cc65
make                        # デフォルト（../src-images/）
make SRC_DIR=../src2        # 別ディレクトリ指定
make MAX_IMAGES=10          # 枚数制限
make GREEN_RESCUE=0         # 緑レスキュー無効
make clean                  # 生成物を削除
```

---

## プレビュー出力

`make` 実行時に各画像の変換結果をソースディレクトリ内の `*_preview.png`（3倍拡大）として出力。
実際のCHRスロット＋ネームテーブルから再構築するため、NES実機出力と一致する。
エミュレータなしで色変換・ディザリング結果を事前確認できる。
